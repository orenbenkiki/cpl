Statically assert that the EXPRESSION does not compile.

The expression should be parameterized by a typename T and should not compile if T is specified to be the TYPE.

This uses some helper types (test::sink and test::sink_t), and some macros (MUST_NOT_COMPILE_WITH_ID, CAT, and CAT2), to force the C pre-processor to generate a unique name for the test class we need to generate for each test.

The implementation is to be blamed on http://stackoverflow.com/questions/23547831/assert-that-code-does-not-compile/23549126#23549126 and is a prime example for what makes C++, well, C++: On the one hand, it is impressive we can implement such a macro. On the other hand, shudder.
