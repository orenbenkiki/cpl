1 CPL {#mainpage}
2 ===
3 
4 <em>"Clever people solve problems wise people avoid".</em>
5 
6 The Clever Protection Library (CPL) tries to provide additional safety for C++
7 projects. For details, see the description of the
8 <code>[cpl](http://orenbenkiki.github.io/cpl/namespacecpl.html#details)</code>
9 namespace in the Doxygen-generated
10 <code>[documentation](http://orenbenkiki.github.io/cpl)</code>.
11 
12 ## Using
13 
14 To use CPL, do the following:
15 
16 - Include `cpl.hpp` in your C++ project.
17 
18 - Use the @ref cpl types instead of the `std` types.
19 
20 - Add one of `-DCPL_FAST` or `-DCPL_SAFE` to your compilation flags.
21  - The `CPL_FAST` variant will compile to code that has no run-time
22  assertions, providing maximal performance.
23  - The `CPL_SAFE` variant will compile to code that has many run-time
24  assertions, providing maximal safety.
25 
26 By convention, a `.fast` or `.safe` suffix is attached to the name of generated
27 libraries and/or binaries to clarify which variant is used.
28 
29 ## Caveats
30 
31 Ideally the `cpl` types would be 100% compatible, drop-in replacements to the
32 `std` types. In practice this is impossible due to the need to protect built-in
33 standard types like `T*` and `T&`. Therefore migration to using CPL requires
34 manually modifying the code.
35 
36 The fast variant is very unsafe. The safe variant is very slow. Achieving
37 simultaneous safety and speed is extremely difficult, and probably impossible
38 in C++. CPL is an engineering compromise. Look into
39 [Rust](http://www.rust-lang.org/) for a new language trying to provide
40 simultaneous safety and speed.
41 
42 The fast variant is as fast as possible. The safe variant is as safe as
43 possible. Obviously it is also possible to define intermediate points on this
44 spectrum, but currently CPL only provides the two extremes.
45 
46 The implementation is naive in many ways. Writing an STL-level library in
47 modern C++ is a daunting task: issues such as `constexpr`, `noexcept` and
48 perfect forwarding are fiendishly difficult to get right. Even simple
49 `const`-ness is tricky due to the existence of both `const opt<T>` and
50 `opt<const T>`.
51 
52 The unit tests are a great help in ensuring everything behaves as it should,
53 however they aren't 100% complete so some edge cases might be wrong. It is also
54 difficult to verify the library forbids all sort of invalid code (even though
55 @ref MUST_NOT_COMPILE helps a bit). In general CPL is relaxed about this
56 because the safe version will detect any foul play at run-time.
57 
58 As always, YMMV, no warranty is implied, may contain nuts, etc.

60 ## Building and Installing
61 
62 CPL is provided as a single header file so there's no building or installation
63 required - just point the include path of the C++ compiler to the CPL
64 directory. However, CPL does come with a `Makefile` providing the following
65 goals:
66 
67 - `make all` - the default goal updates the source files, compiles and runs the
68  tests, and generates the html documentation.
69 
70 - `make src` - updates the source files to embed a version string derived from
71  running `git`, and to enforce a format using `clang-format`. This will not
72  modify the source files timestamp if no changes are required.
73 
74 - `make test` - compiles and runs the tests based on the updated sources.
75 
76 - `make html` - generates HTML documentation using Doxygen based on the updated
77  sources.
78 
79 - `make clean` - removes all the generated files.
80 
81 ## Dependencies
82 
83 There are no dependencies for using CPL, other than, of course, the standard
84 C++ libraries. It was developed using `g++` version 4.9.2 using C++14
85 constructs, and using `clang-format` for formatting the source code. CPL does
86 not currently compile under `clang`, though porting it would make sense.
87 
88 For building and running its unit tests, CPL uses the delightful
89 [Catch](https://github.com/philsquared/Catch) unit tests framework for its
90 tests. Catch is also available as a simple set of header files, the location of
91 which is a configuration parameter in the `Makefile`.
